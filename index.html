GRANDEUR WEB DEVELOPMENT STUDENT PROJECT

Scope of the project and what is required.

  • Have a responsive and beautifull frontend.
  • Probably a button to request details.
  • Probably a button to display details.
  • Use of loops.
  • Conditionals.
  • Two main functions, one to get user details and another to display the user details.
  • The object should be declared globally since both functions make use of it.
  • Console to see errors.

Project

Create a webpage that accepts a user's detail and returns the details on request. The details should be collected using prompts and stored in an object. The details should be validated. The details collected should be username, phone, number, email, password and confirmed password. The details to be displayed on request should contain only the username, phone number, phone number and email.

Validation

  • The username must be less than ten and greater than zero.
  • Phone numbers should be 11 digits.
  • The email address should be 11 digits.
  • Password must be greater than six.
  • User must confirms password. If password and confirm password do not match, alert the user.
